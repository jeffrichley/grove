# Memory Writeback

Use this skill when the work produces durable project knowledge that should be preserved for future runs.

Workflow:
- Read `.grove/GROVE.md` and the relevant `.grove/docs/*` or `.grove/memory/*` surfaces.
- Capture decisions, recurring preferences, and current-state notes in the Grove-owned artifacts.
- Keep writeback concise and factual.

When to avoid:
- Do not store temporary scratch that belongs only to the current step.
